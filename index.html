What is AI?

AI stands for Artificial Intelligence. AI is a type of computer technology that can learn and make decisions, like how humans learn new things by practicing.

But what does that mean? Let's break it down:

  • Artificial = Made by people (not natural)
  • Intelligence = The ability to learn and solve problems.

So AI is technology that can learn from information and use that learning to help people do things better or faster.

Real-World Example: When Netflix suggests shows you might like, that's AI! The Netflix application learned what you enjoy by watching what you've already watched.

AI in Your Daily Life

You probably use AI every day without thinking about it. Here are some common examples:

Here are some common examples of AI:

  • Your phone's autocorrect, which predicts what word you're typing.
  • Voice assistants, like Siri, Alexa, Google Assistant, that understand what you say.
  • Face ID authentication that recognizes your face to unlock your phone.
  • Spam filters to identify junk mail.

All of these use AI to learn patterns and make helpful predictions.

How Do AI Tools Like Copilot Work?

You've learned about AI tools, but how do they actually work? Let's explore the technology behind tools like Microsoft Copilot.

Large Language Models (LLMs)

Copilot and similar AI tools are powered by Large Language Models or LLMs. An LLM is a very sophisticated pattern-matching system that has read millions of books, websites, and code examples.

An LLM is called "large" because it contains billions of parameters. These are tiny pieces of knowledge the model uses to understand and generate text. The more parameters, the more sophisticated the AI can be.

Key Features of LLMs:

  • Pattern Recognition: They identify patterns in how words and code fit together.
  • Context Understanding: They can understand the meaning and context of your questions.
  • Text Generation: They predict what words or code should come next based on patterns they've learned.
  • Versatility: They can handle many tasks without being specifically programmed for each one.

An important point to keep in mind is that LLMs don't "think" or "understand" like humans do. They recognize patterns and make predictions based on the data they were trained on. If there are errors in the training data, then the LLM will make incorrect predictions.

How LLMs Are Trained

Large Language Models learn through a process called training. Here's how it works.

The Training Process

Step 1: Data Collection

The model is given massive amounts of text from books, websites, scientific papers, code repositories, and other sources. This data helps the model learn how language and code work.

Step 2: Pattern Learning

The model analyzes this data to find patterns. For example, it learns that after "The cat sat on the" usually comes "mat" or similar words. For code, it learns that after the keyword "def" in Python there is usually a function name.

Step 3: Adjustment and Refinement

The model makes predictions, checks if they're correct, and adjusts its parameters to improve. This happens millions of times until the model becomes very good at predicting what should come next.

Step 4: Fine-Tuning

After basic training, the model is fine-tuned for specific tasks like coding assistance, having conversations, or answering questions. This makes mode adept at particular jobs.

The Result is a model that can understand your questions and generate helpful responses based on patterns it learned from vast amounts of data.

How LLMs Generate Answers

When you ask Copilot a question, a respone generation process runs behind the scenes.

The Response Generation Process

1. Understanding Your Prompt

The LLM breaks down your question into smaller pieces and analyzes what you're asking for. It identifies keywords and context.

2. Finding Relevant Patterns

Based on your question, the model searches through its learned patterns to find relevant information. For a Python question, it recalls patterns it learned from Python code.

3. Generating the Response

The model generates a response word by word (or token by token). At each step, it predicts what word should come next based on what came before and the patterns it learned.

4. Formatting the Output

The model organizes the response in a clear, readable format and presents it to you.

One important limitation to be aware of is that LLMs don't necessarily have real-time knowledge or access to the internet. Often, they can only use what they learned during training, which means their information has a cutoff date and they can make mistakes or provide outdated information. This is why you need to always check and verify the code that Microsfot Copilot gives you.

AI Isn't Always Right

While Copilot is a useful tool for suggesting code, but it's not perfect at creating code or crafting responses. Sometimes it makes mistakes, called hallucinations.

Copilot might suggest code that looks good but doesn't work or make small syntax errors (like forgetting a colon). Copilot might also misunderstand what you're asking for or give you outdated information. One of the problems junior IT professionals often run into is that Copilot gives them an overly complex solution that's too complicated for what they need.

Your job is to check Copilot's work, just like you'd check a friend's work. You're the programmer - Copilot is just your assistant.

How to Check If Code Is Correct

Whenever Copilot gives you code, follow these three steps:

Step 1: READ It

Do you understand what each line does?

Read through the code carefully. If there's something you don't understand, ask Copilot to explain that specific part. Add comments that explain the code so you'll remember how it works later.

Step 2: TEST It

Does it run without errors?

Try running the code. Does it do what you expected? Does it give you the right output?

Step 3: UNDERSTAND It

Can you explain it to someone else?

If you can't explain how the code works, you don't really understand it yet. Keep asking questions until it makes sense.

Using AI Responsibly

AI is a tool, and like any tool, you need to use it the right way. Here's guidance on using AI responsibly.

‚úÖ DO Use AI To:

  • Help you understand difficult concepts.
  • Get examples of how things work.
  • Learn from well-explained code.
  • Debug errors when you're stuck.
  • Explore new programming techniques.

‚ùå DON'T Use AI To:

  • Copy code without understanding it.
  • Cheat on assignments or tests.
  • Avoid learning. While it may save time, in the long run, you won't gain the skills you need for your career, and that only hurts you.
  • Claim AI's work as entirely your own.
  • Replace your own thinking and problem-solving.

Check Your Syllabus

Keep in mind that different instructors will have different policies about using AI for your coursework, so please check your syllabus for the course policy on AI. For example, while you might be allowed to use a tool like Copilot to generate code examples in a programming course, you may not be allowed to generate an entire research paper for your English course.

‚úÖ Check your syllabus
Does it say anything about AI or Copilot? Some instructors allow it, some don't, and some have specific rules.

‚úÖ Talk to your instructor
Ask questions! Your instructors are here to help you succeed. If you're not sure if you can use a particular AI tool in your course, talk to your instructor about it. They'll appreciate that you're being responsible.

‚úÖ Don't assume AI tools are allowed
Just because AI tools are allowed in one class doesn't mean they are allowed in another. Each instructor has different policies, based on the content and learning outcomes for the course.

Your instructors want to help you! They're not trying to catch you doing something wrong. They want you to learn and succeed. Ask questions and use the resources that are available to you, like the tutoring at the Academic Success Center.

Simple Ways to Cite AI

Example 1: General Help

"I used Microsoft Copilot to help me understand how variables work in Python."

Example 2: Code Assistance

"Microsoft Copilot helped me write the code for the input() function section."

Example 3: Debugging

"I used Microsoft Copilot to help debug an error on line 15."

Check your syllabus to see if there is a specific citation style that your instructor uses in their course, like APA or MLA. These citation styles provide guidelines to make it easy to consistently cite AI tools.

Privacy and Data Security

It's not a good idea to share personal or sensitive information with AI tools! Never share personal information, like passwords, social security numbers, credit card information, addresses and phone numbers, health information or banking information.

Why not?

AI tools that are based on LLMs, like Copilot, learn from interacting with users. They are always training themselves on the data that users are providing. User histories have been leaked. Just as on social media, don't share what you want to keep private.

Common Mistakes to Avoid When Using AI

Let's talk about common errors people make when using AI. This gives you a chance to learn from them so you don't make the same mistakes

‚ùå Mistake #1: Copying Without Reading

Some people just copy Copilot's code without reading it. Then they're surprised when it doesn't work or when they can't explain it.

Solution: Always read and understand the code first.

‚ùå Mistake #2: Not Testing

Assuming the code works without actually running it. AI makes mistakes.

Solution: Always test code before assuming it's correct.

‚ùå Mistake #3: Vague Questions

Asking "python" or "help" instead of being specific.

Solution: Write clear, detailed prompts.

‚ùå Mistake #4: Using AI When You Should Practice

Using Copilot for everything instead of building your own skills.

Solution: Try coding yourself first, use AI when stuck.

‚ùå Mistake #5: Not Asking About Instructor Policies

Assuming it's okay to use AI without checking the syllabus or asking.

Solution: Always check your syllabus and ask your instructor.

The good news? These are all easy to avoid! Just be thoughtful and responsible when using AI.
